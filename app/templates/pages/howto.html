Research project quick start

Research Projects are the basis for everything you do in ioModel. They serve as a way for you to organize your work around outcomes - grouping related tasks and artifacts (data and models).

  1. Using the top nav, click on My Research Projects
  2. Click the New Project button
  3. On the new project screen, give your project set a name and description.
  4. Select the Create button.

Data import quick start

This How-to walks through the process of importing data into ioModel. Once data is imported, it is stored internally in a format that allows for quick access and computation and it is immutable. Any transforms that you apply to the data results in the creation of new data sets.

Once data is imported, descriptive statistics are immediately available and it can be used for correlation analysis, scatter analysis, training models, and creating derived data sets (through sampling, re-coding fields, etc.).

  1. From the My Research Projects page, click the Import Data button
  2. On the import data screen, give your data set a name and description.
  3. Choose a file to import. ioModel supports CSV files with comma delimiters where the first line of the file lists column headings. JSON and direct SQL sampling are planned future features.
  4. Select the Import button.

How to create a classifier

This How-to walks through the process of creating a classifier from a data set that was already imported into ioModel.

  1. From the My Research Projects page, click the hyper-link for the name of the data that the model should be created for.
  2. Click the Train Model button.
  3. Enter a name for your model.
  4. Select Classifier as the model type. Classifiers must be used to predict non-continuous labels. Attempting to do otherwise will result in an error
  5. Select Multiple Linear Regression as the model
  6. Using CTRL or SHIFT select, choose one or more features to train your model.
  7. Select a target column to predict. the target column should not be included in your feature selections.
  8. Press the Train button to finish the model.

How to create a predictor

This How-to walks through the process of creating a predictor from a data set that was already imported into ioModel.

  1. From the My Research Projects page, click the hyper-link for the name of the data that the model should be created for.
  2. Click the Train Model button.
  3. Enter a name for your model.
  4. Select Predictor as the model type. Predictors must be used to predict continuous integer or float data. Attempting to do otherwise will result in an error
  5. Select Multiple Linear Regression as the model
  6. Using CTRL or SHIFT select, choose one or more features to train your model.
  7. Select a target column to predict. the target column should not be included in your feature selections.
  8. Press the Train button to finish the model.

How to evaluate models

When you first train a model using ioModel, the application performs an 80:20 training/test sampling of the data and uses the 80% to train the model and the remaining 20% to evaluate the results. When the model is created, you'll see a different set of measurement results depending on whether you created a predictor or classifier.

Beyond the training/test split evaluation, ioModel supports 2 additional methods for evaluating the performance of your model:

  • Leave one out cross validation (LOOCV) - supported for data sets with an N of < 2000
  • Prediction/evaluation of custom test data sets

To use LOOCV (works well for data with an N < 2000):

  1. From the My Research Projects page, click the hyper-link for the name of the model that you want to evaluate.
  2. Click the Evaluate Model button.
  3. See documentation under Machine Learning for how to interpret the results.

To use custom test data sets (works well for data with an N > 2000):

  1. Using the How to for sampling data, import a larger data set and create a sample of that data.
  2. Using the How to for creating a model, create a model using the desired features and target for the sampled training data set.
  3. Click the Predictions button.
  4. Click the Predict button.
  5. Select the original, larger data set (evident by the data set name and number of rows).
  6. Leave the Prediction Mode value set to default.
  7. Click Next.
  8. See documentation under Machine Learning for how to interpret the results.

How to re-code a field

This How-to explains the process of re-coding a string field with numeric labels. This process is important for creating classifiers as the target labels need to be coded numerically. ioModel makes this process straigtforward by automating all of the boilerplate work.

  1. From the My Research Projects page, click the hyper-link for the name of the data that the model should be created for.
  2. Click the Transform button.
  3. Click the link for Re-code Field.
  4. Enter a name for the derived data set.
  5. Select the field you want to re-code.
  6. Press the Next button.
  7. Enter a numeric value for each label. As a convention, it's good practice to start at 0 and increment forward for each new label
  8. Press the Re-Code button.

Upon completion of the re-coding process, you are taken to a new (derived) data set. The original data is still intact. The new data is saved with the name that was provided and allows provides for a detailed record of each step of processing along with all of the incremental artifacts that were generated.

How to sample from a data set

This How-to explains the process of performing a random sample on an existing data set. This process is important for creating test/training splits of larger data sets or down-sampling a large data set to allow for the use of LOOCV.

  1. From the My Research Projects page, click the hyper-link for the name of the data that the model should be created for.
  2. Click the Transform button.
  3. Click the link for Sample.
  4. Enter a name for the derived data set.
  5. Enter a percent of data you want remaining (.3 for a derived data set containing a random sampling of 30% of the observations).
  6. Press the Sample button.

Upon completion of the sampling process, you are taken to a new (derived) data set. The original data is still intact. The new data is saved with the name that was provided and allows provides for a detailed record of each step of processing along with all of the incremental artifacts that were generated.
